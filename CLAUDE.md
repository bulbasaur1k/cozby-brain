# cozby-brain — project-local instructions

Эти правила переопределяют глобальные для этого проекта.

## Git commits

В этом проекте **можно коммитить напрямую** без отдельной просьбы пользователя.

Правила коммитов:
- Работай в текущей ветке (`main`), без форков/worktree-изоляции/новых веток
- После завершения логической единицы работы — `git add -A && git commit -m "..."` + `git push`
- Сообщения коммитов по conventional commits (feat / fix / chore / docs / refactor / test)
- Без `Co-Authored-By:` и «🤖 Generated with Claude Code» трейлеров
- Если в ходе работы файлы меняются несколько раз — делай один финальный коммит, а не цепочку мелких
- Перед коммитом убедись что `cargo clippy --all-targets -- -D warnings` и `cargo test --workspace` зелёные

Исключения когда НЕ коммитить:
- Незавершённая работа (компиляция падает, тесты не зелёные)
- Пользователь в процессе диалога явно попросил «покажи сначала»
- Эксперимент / временный код без цели сохранения

## Остальное — как в глобальных правилах

Smoke-test перед коммитом обязателен: поднять сервер, один запрос к API, убедиться что бэкенд отвечает.
